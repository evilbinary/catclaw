# CatClaw 核心功能与流程设计

## 一、核心功能

### 1. 简化的智能体运行时
- **嵌入式智能体**：基于轻量级运行时，无需外部依赖
- **工作区**：单一工作目录作为智能体的工作空间
- **启动文件**：简化的启动文件注入（AGENTS.md、SOUL.md）
- **技能系统**：基础技能加载机制

### 2. 基础会话管理
- **会话键**：简单的会话标识机制
- **会话存储**：本地文件存储会话历史
- **会话重置**：基础的会话重置机制

### 3. 核心消息处理
- **消息路由**：简单的消息路由机制
- **队列系统**：基本的消息队列处理
- **流式传输**：基础的消息流式传输

### 4. 基础工具系统
- **核心工具**：read、exec、write 等基础工具
- **工具执行**：简化的工具执行流程

### 5. 简化记忆管理
- **记忆文件**：基础的 Markdown 记忆文件
- **记忆检索**：简单的记忆检索功能

### 6. 基础模型集成
- **模型配置**：简单的模型配置机制
- **模型调用**：基础的模型调用流程

## 二、核心流程

### 1. 系统启动流程
1. 加载配置文件
2. 初始化智能体运行时
3. 加载工作区文件
4. 启动消息监听

### 2. 消息处理流程
1. 接收入站消息
2. 确定会话键
3. 检查队列状态
4. 构建系统提示
5. 调用模型
6. 处理模型响应
7. 执行工具（如果需要）
8. 发送出站消息

### 3. 会话管理流程
1. 会话创建：新消息触发新会话
2. 会话存储：将对话历史写入文件
3. 会话重置：基于时间或手动触发

### 4. 记忆管理流程
1. 记忆写入：将重要信息写入记忆文件
2. 记忆读取：在会话开始时加载记忆
3. 记忆检索：根据需要检索相关记忆

### 5. 工具执行流程
1. 解析工具调用请求
2. 执行工具操作
3. 处理工具执行结果
4. 将结果返回给模型

## 三、架构设计

### 1. 核心组件
- **Gateway**：简化的消息网关
- **Agent Runtime**：轻量级智能体运行时
- **Session Manager**：会话管理组件
- **Memory Manager**：记忆管理组件
- **Tool Executor**：工具执行组件
- **Model Connector**：模型连接组件

### 2. 数据流
```
消息 → Gateway → Session Manager → Agent Runtime → Model Connector
                                      ↓
                               Tool Executor
                                      ↓
                               Memory Manager
```

## 四、配置与部署

### 1. 配置文件
```json
{
  "agent": [{
    "name": "default",
    "baseUrl": "https://api.openai.com/v1",
    "model": "openai/gpt-3.5-turbo",
    "apiKey": "YOUR_API_KEY"
  }],
  "works":{
    "workspace": "~/.catclaw/workspace",
    "agent": "default"
  },
  "session": {
    "reset": {
      "mode": "daily",
      "atHour": 4
    }
  },
  "tools": {
    "allow": ["read", "exec", "write"]
  },
  "debug": true
}
```

### 2. 部署步骤
1. 安装依赖
2. 配置 API 密钥
3. 初始化工作区
4. 启动服务

## 六、技术栈选择

### 1. 后端
- Node.js：轻量级运行时
- Express：简单的 HTTP 服务器
- SQLite：本地数据存储

### 2. 前端
- 命令行界面：基础交互
- 简单的 Web 界面：可选

### 3. 模型
- OpenAI API：默认模型
- 其他模型：可扩展

## 七、简化策略

1. **减少通道支持**：仅支持最常用的通道（如命令行、Web 界面）
2. **简化会话管理**：使用简单的文件存储，减少复杂的会话状态管理
3. **精简工具集**：仅保留最核心的工具，减少工具复杂度
4. **简化记忆系统**：使用基础的文件存储，减少向量搜索等复杂功能
5. **减少配置选项**：提供合理的默认值，减少配置复杂度

## 八、扩展性考虑

1. **模块化设计**：核心组件模块化，便于扩展
2. **插件系统**：基础的插件机制，支持功能扩展
3. **模型切换**：支持不同模型的切换
4. **通道扩展**：支持添加新的消息通道

这个 mini 版本保留了 OpenClaw 的核心功能和设计理念，同时大大简化了实现复杂度，适合快速部署和学习使用。
